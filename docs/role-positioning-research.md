## The three roles — same archetype, different emphasis

```
                    DESIGN INTENT
                         │
         ┌───────────────┼───────────────┐
         │               │               │
    Prototype &      Design Systems    Ship & Polish
    prove vision     & production UI   end-to-end
         │               │               │
         ▼               ▼               ▼
  Design Technologist  UX Engineer   Design Engineer
  (Amazon, innovation) (Google, Meta) (Vercel, Cursor, Apple)
         │               │               │
         └───────────────┴───────────────┘
                         │
                    SHIPPED SOFTWARE
```

| Dimension | Design Engineer | UX Engineer | Design Technologist |
|-----------|-----------------|-------------|---------------------|
| **Primary examples** | Vercel, Cursor, Apple (product design eng) | Google, Meta, enterprise UXEP-style teams | Amazon (Devices, Prime, Ads Innovation Lab) |
| **Core superpower** | Own outcomes — design, build, ship with craft | Bridge disciplines; prototype *and* productionize | Prove ambitious ideas through functional prototypes |
| **Typical output** | Production UI, marketing surfaces, design system docs | Reusable components, research prototypes, UX libraries | Demos, design system libraries, 0→1 validation |
| **Handoff model** | Eliminates it — iterates in Figma *or* code | Translates both ways; "production-izes" prototypes | De-risks before engineering commits |
| **Expert signal** | Taste + systems + speed; pixel-perfect + performant | Speaks both languages; reduces translation loss | Makes the impossible feel real; influences without authority |

## Company-by-company findings

### Vercel — Design Engineer

**Source:** [Design Engineering at Vercel](https://vercel.com/blog/design-engineering-at-vercel)

- Blend aesthetic sensibility with technical skills; understand, design, build, ship autonomously.
- Skip traditional handoff — iterate in Figma or code from idea to production.
- Care about delight, reusable primitives, page speed, accessibility, cross-browser polish.
- No fixed toolset or "correct" background; team diversity and good taste matter.
- Collectively capable of: Figma, code, production engineering, performance debugging, shaders, 3D, copy, video.
- **Hiring note:** Apply even without the Design Engineer title.

**Keywords for portfolio:** polish, performance, accessibility, design systems, skip handoff, iterate to greatness.

### Cursor (Anysphere) — Design Engineer

**Source:** [cursor.com/careers/design-engineer](https://cursor.com/careers/design-engineer)

- Lives at intersection of design and code; works with founding team.
- Production-quality UI, rapid prototypes (hours not days), pixel-perfect implementation.
- TypeScript, React, SolidJS, CSS, animations; design systems contribution.
- Roles intentionally muddy — designers prototype in Cursor, not only Figma.
- Bonus: developer tools, Electron, code editors, AI product passion.

**Keywords for portfolio:** speed, systems thinking, developer tools, live prototypes, muddy roles.

### Google — UX Engineer (UXE)

**Sources:** [UX Engineering at Google](https://uxe.withgoogle.com/), [Why Google Needs UX Engineers](https://design.google/library/why-google-needs-ux-engineers)

- "Amorphous unicorns" at intersection of design and engineering.
- **Design lens:** prototypes for research, feasibility, leadership reviews.
- **Engineering lens:** production code, design systems, Core Web Vitals, security, test coverage.
- Prototyping is the throughline — "a prototype is worth a thousand meetings."
- Acts as translator between designers and engineers; production-izes prototypes.
- Traits: action-biased, empathetic communicator, adaptive, inquisitive.

**Keywords for portfolio:** prototypes, design systems, bridge role, production-ize, reusable components.

### Meta — Design Engineer (software)

- Validate hypotheses with advanced prototypes; 0→1 concepts for research and leadership.
- Improve design systems, tools, quality/craft initiatives.
- Full lifecycle: ideation → prototype → validation → user research.
- Portfolio must show creative use of code across web and native.
- *Note:* Meta also uses "Design Engineer" for hardware/mechanical roles — software DE is distinct.

**Keywords for portfolio:** interactive prototypes, craft initiatives, research-ready builds, cross-platform.

### Amazon — Design Technologist

**Sources:** Amazon.jobs (Devices & Services, Prime UX, Agentic AI), [Fitz Maros career story](https://advertising.amazon.com/careers/re-exploring-careers/fitz-maros-insights-on-becoming-design-technologist)

- Build functional prototypes that prove and advance product vision.
- Design systems at scale (e.g. Prime Design System).
- Bridge creative and technical stakeholders; translate jargon both ways.
- 3–10+ years front-end/prototyping; portfolio + code samples required.
- Communication paramount; diverse backgrounds valued.

**Keywords for portfolio:** functional prototypes, innovation, design system library, customer obsession, ambiguity.

### Apple — Design Engineer (within design org)

- Infrastructure for designers to build in code (Swift, SwiftUI, web).
- CI/CD, TestFlight, version control workflows for design teams.
- Elevate prototypes to structured, adoptable artifacts for production teams.
- AI-assisted development changing what's possible for design orgs.

**Keywords for portfolio:** designer-enablement, prototype infrastructure, Swift/web hybrid, maintainable code.

## Why these roles are in high demand (2025–2026)

1. **Handoff collapse** — AI and modern stacks compress decide → ship. Static mockups lose to working prototypes and production code.
2. **Bottleneck shift** — Code generation commoditizes; taste, systems thinking, and judgment become scarce.
3. **Translation loss is expensive** — One person with closed-loop ownership beats designer + engineer + PM chains for speed and fidelity.

Michelle's timing aligns with market naming and hiring for work she's already doing.

## Recommended portfolio strategy

### Primary identity

**Design Engineer** — highest market momentum (Vercel, Cursor, AI-native companies); matches trajectory; portfolio site itself becomes proof.

### Secondary frames (contextual)

- **UX Engineer** — Google, Meta, large product orgs, design-system-heavy teams.
- **Design Technologist** — prototyping, innovation, 0→1 concept stories.

### Tone balance

| Path A | Path B |
|--------|--------|
| Design Engineer energy (startup / AI-native) | Enterprise credibility |
| Fast, polished, artifact-forward | Complex, systemic, strategic |
| Portfolio = product | Case studies show depth |

**Lead with Path A in headline and visual energy; carry Path B in case study depth.**
