# UX Enablement Platform

> Slug: `case-studies/uxep-platform/` · Status: draft · Visibility: internal

## One-liner

Designed and shipped an enterprise UX enablement platform, from information architecture and high-fidelity design through production release, documentation migration, and analytics instrumentation.

## Role

- **Title on project:** Senior Product Designer
- **Scope:** design + production code + systems

## Context

- **Organization:** Cisco (IT Design / UX Enablement)
- **Timeline:** Feb 2026 – May 2026 (v1 launch); v2 exploration ongoing in `drafts/uxep-v2/`
- **Team:** Jarod, Caroline, Nixon, Alex, Brian, Josh, Theresa, UXEP dev squad

## Problem

UX enablement content and tooling were fragmented across Confluence and legacy surfaces. Designers and developers needed a single, design-system-aligned home for guidance, with measurable adoption and a build process that could keep pace with AI-assisted delivery.

## Approach

- Synthesized proposed v2 IA from Confluence and live site (see `drafts/uxep-v2/` for continuation)
- High-fidelity work on homepage bands (impact stats, testimonials, FAQ, editorial panels)
- Documentation lift-and-shift (CDSI, templates, developer docs, data visualization, release notes)
- OpenSpec-driven build process with engineering; CI/CD documentation for the team
- Analytics: WARP metrics gap analysis, tagging onboarding for atmosphere.cisco.com, see `supporting/metrics-and-analytics/`

## Outcome

- **Shipped:** Production release May 18, 2026
- **Recognition:** UXEP ranked #2 on AI-Native leadership board
- **Engineering:** Multiple PRs authored and reviewed across release readiness
- **Analytics:** Tagging onboarding merged; WARP access and metrics workflow established
