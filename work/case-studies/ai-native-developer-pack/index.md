# AI-Native Developer Pack

> Slug: `case-studies/ai-native-developer-pack/` · Status: draft · Visibility: internal

## One-liner

Led design and discovery for a five-day POC that puts manifest-driven AI tool discovery and onboarding in the IDE, from sharded YAML catalog to a working CDSI/Atmosphere vertical slice.

## Role

- **Title on project:** Senior Product Designer (scrum lead for POC week)
- **Scope:** design + prototype + production code

## Context

- **Organization:** Cisco (IT Design / AI-Native Engineering)
- **Timeline:** June 2026 (POC); MPP pivot July 2026
- **Team:** AI-Native Dev Pack squad

## Problem

Cisco AI tooling was hard to discover and activate in context. Teams needed deterministic, touchstone-driven setup for project onboarding, not implicit advertising of tools, with consent-based activation through APM.

## Approach

- Facilitated five-day POC build: discovery skills, CDSI/Atmosphere vertical slice, validation, packaging, demo hardening
- Authored 8 PRs / reviewed 18 PRs on `cisco-it-design/ai-native-developer-pack`
- Documented POC vs. future state, demo script, and explicit demo flow (React + ATM DS → Pomodoro app)
- Pivoted MPP toward deterministic Project Setup and Day One Onboarding flows

## Outcome

- **POC proven:** Manifest-driven discovery → domain scoring → progressive skill disclosure → composed install commands
- **Demo:** Vidcast recorded; explicit demo flow documented
- **Next:** MPP scope defined with assignees while transitioning from POC

**Elevator pitch:** The AI-Native Developer Pack puts discovery and onboarding where you already work, in the IDE. With one `apm install` and guided activation it suggests relevant tools during engineering work, explains why they match, and activates ready tools through APM after you consent.
